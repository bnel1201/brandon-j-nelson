## Writing workflow with Markdown and Pandoc

I used to enjoy writing physics and math papers using LaTeX in undergrad but being a mostly Windows user always struggled with getting an environment set up, even with complete packages like [TexLive](https://tug.org/texlive/). Additionally my lab group where I do my graduate research primarily use Microsoft Word so I'd have to keep seperate versions of everything in TeX and word which wasn't practical.

I've really gotten into [Pandoc](https://pandoc.org/) for writing [my thesis]({{ site.baseurl }}/thesis/) and lab notebook using [Markdown](https://en.wikipedia.org/wiki/Markdown) which has largely addressed these challenges of setting up an environment and sharing files with colleagues in their preferred file formats. I'm certainly not the first to do this but the motivation for doing so while exploring note taking strategies (this topic is worthy of a blog post on its own!) for my lab notebook (which absolutely had to be electronic due to my awful handwriting!). Again, like many before me I came to really enjoy [VS Code](https://code.visualstudio.com/) as my do-it-all text editor, especially for its [Markdown extensions](https://marketplace.visualstudio.com/items?itemName=yzhang.markdown-all-in-one) which enabled me to hop around between markdown files using [Markdown links](https://www.markdownguide.org/basic-syntax/#links). Hyperlinks are the most important part of note taking for me, because they allow me to break up my notes into bite-sized chunks to avoid being overwhelmed while also maintaining connections between ideas. I also liked having this feature while writing my thesis because I could break up my chapters into different files while quickly hopping back and forward.

I have more detailed description of some of the tools I made to help with this in this [post on markdown-publish]({{ site.baseurl }}/markdown_publish/)
